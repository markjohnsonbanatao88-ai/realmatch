# Privacy and Data Retention

This static MVP does not collect, store, process, or transmit real applicant data.

## Principles

- Collect only what is required for assessment, verification readiness, member safety, consent records, and operations.
- Do not collect sensitive information unless there is a documented business need and lawful basis.
- Do not enable identity-document upload until secure storage, access controls, review logs, and deletion logs exist.
- Keep role-based access narrow.
- Keep retention periods documented.

## Identity document rule

Identity documents must not be retained longer than the approved verification window unless there is a documented legal reason.

Required record fields:

- member or applicant ID
- document type reviewed
- reviewer
- review timestamp
- result
- deletion timestamp
- deletion proof reference
- exception reason if retained

## Pre-launch requirement

A lawyer-reviewed privacy notice, retention schedule, and data subject request process must exist before collecting real personal data.
