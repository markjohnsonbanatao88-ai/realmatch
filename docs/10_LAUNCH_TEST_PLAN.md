# Launch Test Plan

Current status: static MVP only. Do not run live acquisition until compliance, privacy, and operational owners are confirmed.

## Goal

Validate whether the positioning, application flow, and pricing can generate serious qualified interest without creating legal, safety, or reputational risk.

## Test audience

Start with warm or permission-based leads only. Avoid mass outreach until consent source, opt-out workflow, and message review are documented.

## Funnel

1. Landing page visit.
2. Private assessment view.
3. Application completion.
4. Manual review.
5. Review call.
6. Paid consultation readiness.
7. Optional coordination request after mutual interest.

## Metrics

- visits
- application starts
- application completions
- accepted applicants
- rejected applicants
- review completions
- paid consultation conversions
- coordination requests
- complaints
- opt-outs
- refund requests

## Go criteria

- Serious applicants understand the platform boundary.
- No pressure-based or exploitative expectations dominate demand.
- Staff can process reviews without improvising policy.
- Safety and complaint workflows are understood.

## Stop criteria

Pause the test if leads repeatedly ask for guaranteed outcomes, staff drift into prohibited positioning, or member safety controls are not ready.
