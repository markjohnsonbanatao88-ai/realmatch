# Next Implementation Roadmap

## Phase 1: Static MVP

Complete public pages, application UI, admin mock, compliance docs, and CI validation.

## Phase 2: Backend applications

Add secure server-side application submission, validation, and storage.

## Phase 3: Admin authentication

Add role-based admin login, authorization, and access logs.

## Phase 4: Review workflow

Add secure review records, deletion logs, reviewer ownership, and status transitions.

## Phase 5: Payment links

Add manual payment-link tracking for approved applicants only. Keep fee labels aligned with consultation and coordination services.

## Phase 6: Member discovery

Add member-controlled visibility and independent interest expression. Do not add assignment logic.

## Phase 7: Coordination workflow

Add optional scheduling and venue coordination only after mutual interest and consent records exist.

## Phase 8: Audit logging and reporting

Add immutable audit logs, complaint reports, consent exports, opt-out records, and operational dashboards.

## Rule

Do not skip phases. Scale after evidence and process discipline, not before.
